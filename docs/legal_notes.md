# Legal & Ethical Notes

> Disclaimer: this document is operational guidance for a research/educational
> project. It is **not** legal advice. If you ever publish this product or
> monetize it, consult a Vietnamese IP/internet-law lawyer.

## 1. Project framing

`vn-news-summarizer` is an **educational and research project**. It exists
to study Vietnamese news summarization with NLP techniques, not to compete
with publishers, redistribute their content, or replace visiting their sites.

## 2. What we do (and don't do) with crawled content

**We do:**
- Fetch publicly available RSS feeds and article HTML.
- Extract main text into our database for processing only.
- Display **short summaries (≤ ~70 words, 2–3 sentences)** generated by our
  models, alongside the **original title**, **source name/logo**, **publish
  time**, and a **prominent link to the original article**.
- Cache content for the minimum time needed to operate (target: **≤ 30 days**
  of full text retention; configurable cleanup).

**We do not:**
- Republish full articles.
- Strip source attribution.
- Bypass paywalls, login walls, or subscriber-only sections.
- Serve our own ads next to summaries (project is non-commercial).
- Train any model on text from sources whose ToS forbid it.

## 3. Crawling discipline

- **`robots.txt`** is parsed and obeyed for every host before any HTTP
  request. If a path is disallowed, we skip it.
- **User-Agent** is honest and identifies the project + a contact URL:
  `vn-news-summarizer-research/0.1 (+https://github.com/khangnh22ds/vn-news-summarizer)`.
- **Rate limit:** ≤ 1 request per second per host with jitter. Exponential
  backoff on 429 / 5xx responses.
- **RSS first.** HTML scraping is a fallback and used only when RSS does
  not surface required metadata.
- **Sitemap.xml** allowed but only as a discovery mechanism, never to scrape
  paywalled sections.

## 4. Source attribution (UI requirements)

Every rendered article card MUST include:
- Original title (visible).
- Source name and (where available) logo, linked to source homepage.
- Publish timestamp.
- "Đọc bản gốc" button linking to the original URL with
  `target="_blank"` and `rel="noopener noreferrer nofollow"`.
- A small label "Tóm tắt bởi mô hình AI — có thể chưa chính xác" under
  the summary.

The About page lists every source with a link and a takedown contact.

## 5. Takedown / opt-out

- A public contact (email or GitHub issue) is published in the README and
  About page so any publisher can request removal of their source.
- On takedown request: disable the source in `configs/sources.yaml`,
  delete its rows from `articles`, `summaries`, `labels`, `crawl_runs`
  within 7 days, and document the action in `docs/takedowns.md` (created
  on first use).

## 6. LLM labeling & training data

- Summaries used for training are **derived** outputs (≤ 70 words),
  not direct copies.
- Training datasets are not redistributed. They live in the repo's
  gitignored `data/processed/` and are versioned via DVC for our use.
- If we ever publish a dataset for research, we will publish only:
  - URLs + metadata,
  - our generated summaries,
  - a script to reproduce the article fetch from the URL,
  i.e., we never redistribute publishers' full text.

## 7. Vietnamese context

- Luật Sở hữu trí tuệ (Vietnamese IP Law) recognizes the right of authors
  and press organizations over their works. There is no clear
  "fair use" doctrine; courts have not ruled specifically on LLM
  summarization. The project's posture is: short transformative
  summaries with source attribution and link-out is a **conservative
  best-effort interpretation**, not a guaranteed safe harbor.
- Avoid scraping content marked "Bài viết được bảo vệ" or paywalled.
- Do not crawl personal data or private forums.

## 8. Privacy

- The MVP has **no user accounts** and stores no user-identifying data
  beyond standard server access logs (≤ 30 days).
- If accounts are added later, follow Decree 13/2023/ND-CP (Vietnam
  personal data protection) and add a Privacy Notice page.

## 9. Operational checklist before each deploy

- [ ] `robots.txt` checks still pass for every enabled source.
- [ ] No source has flagged a takedown that hasn't been honored.
- [ ] UI shows attribution + AI disclaimer for every card.
- [ ] Rate limits in `.env` match the values declared here.
- [ ] Data retention cron is scheduled.
